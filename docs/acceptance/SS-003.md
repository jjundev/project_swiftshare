# SS-003 acceptance evidence

## Automated commands

```sh
make test
```

## Implemented

- Separate bounded Pairing v1 Protobuf contract, QR envelope, cross-language QR,
  transcript, comparison-code, wire-codec, failure-budget, race, and expiry tests.
- Mac TLS listener and QR UI using the provisioned non-exportable Keychain P-256
  identity, exact certificate fingerprint, Android proof verification, local
  confirmation, durable Peer record, signed receipt, and Android acknowledgment.
- Android CameraX plus bundled offline ML Kit QR scanning, exact Mac certificate
  pinning, Keystore proof, bilateral confirmation, durable Peer record, and paired
  success UI.
- Paired Peer lists and local unpair on both clients. Android TLS receive mode can
  use the live Paired Peer repository instead of a manually entered pin.
- Missing Android Keystore identity or missing Mac Keychain identity clears stale
  local Peer records. Changed remote keys fail pinned authentication.
- Unpair during a pre-Commit fixed-endpoint transfer uses authenticated
  `peer_unpaired` cancellation and prevents subsequent admission.

## Required real-device observations

- QR scan and bilateral comparison-code confirmation on one named Mac and Android
  14+ Device on an offline LAN.
- Wrong fingerprint, expiry, rejection, replay, concurrent second scan, key deletion,
  unpair, and unpair-during-transfer behavior.
- A newly paired identity completes SS-002 with manual/seeded trust disabled.

Real-device observations remain hardware-gated and are not claimed by automated
builds.
