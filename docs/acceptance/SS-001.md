# SS-001 acceptance evidence

This file records reproducible checks for SS-001. It is updated with observed
results rather than screenshots or generated build output.

## Clean-workspace definition

A clean-workspace check starts after `make clean` and verifies that ignored build
outputs are absent. Dependency caches outside the repository may remain populated;
the source tree itself must not depend on generated files that are absent from a
clean checkout.

## Commands

```sh
make doctor
make build-macos
make test-macos
make build-android
make test-android
SWIFTSHARE_ANDROID_SDK=/definitely/missing ./scripts/doctor android
```

## Required observations

- The macOS development app builds, launches as a menu-bar-only application, and
  has `LSUIElement` enabled.
- Swift tests read `contracts/fixtures/foundation-v1.json` and exercise the
  Transfer Session seam.
- The Android development app builds and launches on an API 34+ emulator or device.
- Kotlin tests read the same fixture and exercise the same Transfer Session seam.
- An explicitly invalid Android SDK path produces a non-zero, actionable diagnostic.
