# SS-004 acceptance evidence

## Automated command

```sh
make test
```

## Implemented

- Version-neutral, bounded Hello/Decision bootstrap with shared Swift/Kotlin
  Protobuf byte vectors and semantic outcome fixtures.
- Highest-common major/minor negotiation, version-scoped named capabilities,
  unknown optional compatibility, required-capability rejection, and common Chunk
  selection before Manifest.
- Device-wide tokenized schedulers in Swift and Kotlin with ordinary busy,
  stable SPKI-key-ID tie-break, loser draining state, conditional release, and
  subsequent new-session retry tests.
- Mac fixed-endpoint sender sends Hello first, validates the receiver selection,
  and binds all later record headers and Chunk slicing to the negotiated values.
- Android fixed-endpoint receiver uses a separate acceptor and bounded workers,
  revalidates repository trust before admission, returns authenticated bootstrap
  rejection, and reaches approval or MediaStore reservation only after acceptance.
- User-visible busy and incompatible-version categories without replacing an
  already active receiver UI with a rejected contender.

## Required real-device observations

- Compatible Mac-to-Android file transfer after upgrading both development builds.
- Major mismatch and required-capability mismatch display an upgrade error on both
  Devices and allocate no pending MediaStore item.
- An active transfer returns authenticated busy to a second paired connection.
- Actual reverse-direction transport and a two-real-adapter simultaneous start are
  certified with SS-007; SS-004 certifies their shared scheduler contract.
