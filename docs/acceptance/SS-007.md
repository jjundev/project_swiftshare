# SS-007 acceptance evidence

## Automated commands

```sh
make test
xcodebuild -project macos/SwiftShare.xcodeproj -scheme SwiftShare \
  -configuration Debug -destination 'platform=macOS' CODE_SIGNING_ALLOWED=NO build
cd android && ./gradlew --no-daemon :app:compileDebugKotlin
```

## Implemented

- Android single-document preparation with persistable URI permission, atomic draft
  persistence, bounded-buffer hashing, relaunch rehydration, metadata/readability
  revalidation, and streamed digest/length refusal as `source_changed`.
- Android reverse DNS-SD browser that hides candidates until pinned mutual TLS and a
  nonce probe identify a current Paired Peer, plus a fresh authenticated connection
  for the Transfer Session.
- Android sender review, Peer selection, approval wait, progress, cancel, and terminal
  UI; outbound work cancels when the visible Activity leaves the foreground.
- Production Kotlin outbound and Swift inbound roles using the existing v1 records,
  typed terminal outcomes, deadlines, cancellation, final digest verification, and
  the shared device-wide scheduler.
- Mac writable security-scoped bookmark persistence and renewal, destination
  preflight, same-directory hidden staging, durable recovery journal, file/directory
  synchronization, exclusive atomic Commit, and deterministic numeric suffixes.
- Mac reverse listener and DNS-SD advertisement gated by current destination and
  paired trust, plus receiver approval, progress, cancel, terminal UI, and no local
  filesystem path disclosure on the wire.
- Follow-up hardening serializes Android outbound state and send admission, releases
  newly acquired URI grants on failed preparation, gates overlapping Mac listener
  restarts by generation, rejects zero-length Chunks, and explicitly shows the
  single-item review count.
- Regression tests cover duplicate send admission, independent state-field updates,
  URI-grant rollback, latest restart generation, deterministic bounded suffixes, and
  zero-length Chunk refusal, plus idempotent abandoned-stage journal reconciliation
  in the Xcode adapter test target.

## Required real-device observations

- Select a provider-backed Android document, kill/recreate the process, and confirm
  the draft remains readable and review metadata is unchanged.
- Transfer on one supported physical Android Device and sandboxed signed Mac, both
  with manual approval and with per-Peer auto-accept.
- Exercise sender and receiver cancellation, source permission revocation, source
  mutation, destination bookmark revocation, full destination, network loss, Chunk
  corruption, and final digest corruption; verify no partial final item appears.
- Pre-create the requested filename and race two independent exclusive allocations;
  verify existing files remain byte-identical and deterministic suffixes are used.
- Transfer a multi-gigabyte document and record bounded memory on both Devices.
- Relaunch the Mac after an interrupted staged transfer and verify the hidden stage is
  reconciled before reverse availability resumes.
