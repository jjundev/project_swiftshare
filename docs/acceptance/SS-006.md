# SS-006 acceptance evidence

## Automated commands

```sh
make test
```

## Implemented

- Receiver-owned `PeerApprovalPolicy` in the Kotlin and Swift domain seams; sender
  requests no longer carry receiver policy.
- Schema-v2 Android and macOS Peer persistence with default manual approval,
  separate trust/policy revisions, semantic identity validation, and policy reset
  on unpair.
- Android process-scoped receive runtime with app-open-only default, explicit
  connected-device foreground Service, ongoing notification, immediate Stop,
  notification-permission gating, network epoch restart, and persisted reversible
  availability choice.
- Android auto-accept after pinned TLS, compatibility negotiation, endpoint-ticket
  validation, final trust validation, scheduler admission, and Manifest parsing.
- Peer-specific unpair cancellation rather than cancellation of an unrelated active
  session.
- Mac per-Peer auto-accept controls, system-authoritative `SMAppService.mainApp`
  launch-at-login control, resilient multi-route racing, and a five-second overall
  route-resolution deadline with a typed unavailable error.
- Cross-language domain tests proving auto-accept bypasses only the manual approval
  gateway, plus Android instrumentation coverage for availability persistence and
  trust/policy revision independence.

## Required real-device observations

- Android app-open-only stops on background and screen lock and succeeds after the
  app returns to the foreground.
- Receive Mode remains discoverable while backgrounded and locked, shows its ongoing
  notification, opens manual incoming review, and stops immediately from its action.
- Notification denial/disable, Task Manager Stop, ordinary process reclamation,
  Wi-Fi loss/reconnect, address change, and OS foreground-service restriction each
  produce the documented state.
- Mac-to-Android transfer succeeds once with manual approval and once with per-Peer
  auto-accept, then fails within five seconds after Android becomes unavailable.
- Launch-at-login is exercised from a stable signed development app location and its
  enabled, disabled, and requires-approval states are observed.

Production Mac inbound payload reception and destination Commit remain SS-007.
