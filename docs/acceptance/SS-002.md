# SS-002 acceptance evidence

This file records reproducible checks for the fixed-endpoint Mac-to-Android file
transfer. Evidence is added only after the corresponding command or real-device
scenario has been observed.

## Automated commands

```sh
make test-macos
make test-android
```

## Required real-device setup

- A named Mac running macOS 14 or newer.
- A named Android Device running Android 14/API 34 or newer.
- Both Devices on one LAN with internet connectivity disabled for the transfer.
- An ignored development configuration containing the Android address, default port
  `8443`, public certificate/SPKI pins, and no private key material.

## Required observations

- Authenticated approval and one-file completion under Downloads/SwiftShare.
- Sender and receiver phase plus byte/item progress.
- Sender rejection, Mac cancellation, Android cancellation, network failure, Chunk
  corruption, final digest corruption, permission failure, and storage failure.
- No visible completed item before verification or after failure.
- Existing names remain unchanged and SwiftShare-concurrent allocations receive a
  deterministic numeric suffix.
- A multi-gigabyte transfer remains bounded in memory. The later certification
  target is below 150 MiB per application; SS-002 records the observed value rather
  than treating that future hardening threshold as its completion gate.

## Current observations

- Swift and Kotlin independently pass the same v1 header, Chunk prelude, control
  message, fragmentation, truncation, oversize, malformed-message, and role-session
  tests.
- The signed/sandboxed Mac development app builds with its fixed-endpoint sender,
  Keychain identity lookup, TLS 1.3 pin verification, file pre-hash, bounded Chunk
  streaming, progress, and cancellation UI.
- The Android debug APK builds with its foreground-only TLS 1.3 listener, Android
  Keystore identity, manual approval UI, ordered Chunk verification, MediaStore
  pending reservation/Commit, deterministic suffix allocator, and cleanup paths.
- Android instrumentation sources compile for Keystore non-exportability and
  MediaStore pending/publication behavior.
- Real-device reference, LAN endpoint, instrumentation execution, and offline E2E
  observations are not yet available.
